# App

컨테이너 이미지로 빌드될 애플리케이션 영역입니다.

현재는 CI 파이프라인 구조를 보여주기 위한 최소 정적 페이지와 Dockerfile만 포함합니다. 이후 실제 백엔드, 프론트엔드, API 서버 또는 샘플 마이크로서비스로 교체할 수 있습니다.

## 확장 가이드

- 런타임을 선택합니다: Node.js, Python, Go, Java 등
- `/healthz` 같은 헬스 체크 엔드포인트를 추가합니다.
- Dockerfile을 실제 애플리케이션 빌드 방식에 맞게 교체합니다.
- Kubernetes `deployment.yaml`의 포트, 환경 변수, 리소스 설정을 애플리케이션에 맞게 갱신합니다.
